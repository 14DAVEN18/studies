Operador nodeBindCallback

Convierte la respuesta de una API Callback de tipo Node.js en un observable rxjs para manejar la respuesta de manera más flexible.

Es tal como un bindCallback pero se espera que el callback sea de tipo callback(error, result).

bindCalback no es un operador porque sus entradas y salidas no son observables. La entrada es una función func con algunos parametros pero el último parametro debe ser una función callback que func llama cuando termina. Se espera que la función callback siga las convenciones Node.js, en donde el primer argumento del callback es un objeto error que señala si la llamada fue exitosa. Si el objeto es pasado al callback, esto significa que algo salió mal.

Ejemplo: Se usa el operador ajax para hacer una petición HTTP de tipo GET a una url, cuando el servidor responde, el método devuelve la respuesta envuelta en un observable y agrega un delay de 2000 ms, esto permite a la aplicación procesar la información de manera reactiva.
